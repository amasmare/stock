## سير العمل مع قائمتك (2604 سهم)
ملف `tickers 2604.csv` هو **قائمة رموز** فقط (Symbol,Name,Sector,Market_Cap)،
لذا أولاً ننزّل تاريخ الأسعار ثم نمسح:
```bash
pip install yfinance

# 1) تنزيل أسعار أكبر 100 سهم بالقيمة السوقية (سنتين)
python fetch.py "tickers 2604.csv" --limit 100 --period 2y

# 2) المسح بأنماط إليوت
python scan.py ./data --min-score 65 --csv-out results.csv

# فلاتر مفيدة عند التنزيل:
python fetch.py "tickers 2604.csv" --sector Technology --min-cap 10 --limit 50
python fetch.py "tickers 2604.csv" --symbols AAPL,MSFT,NVDA,LLY
```

## التشغيل السريع (بيانات تجريبية)
```bash
# 1) بيانات تجريبية للاختبار
python make_sample.py

# 2) مسح المجلد (القوانين المكتملة فقط، score >= 55)
python scan.py ./data

# 3) كل الأنماط حتى غير المكتملة
python scan.py ./data --all --min-score 0

# 4) فلترة نوع واحد
python scan.py ./data --pattern Impulse --min-score 70

# 5) تفاصيل سهم واحد (قوانين + فيبو + أهداف)
python scan.py ./data --ticker AAPL

# 6) حفظ النتائج
python scan.py ./data --csv-out results.csv
```

## صيغة ملفات CSV
كل ملف = سهم واحد، واسم الملف = رمز السهم (مثل `AAPL.csv`).
الأعمدة المطلوبة (غير حساسة لحالة الأحرف):
```
Date,Open,High,Low,Close,Volume
2023-01-03,125.07,130.90,124.17,125.07,112117500
```
- **الأعمدة الإلزامية:** `High, Low, Close`. البقية اختيارية.
- البيانات اليومية أو الأسبوعية تعطي أفضل نتائج لإليوت.

## كيف يعمل
1. **ZigZag** يكتشف القمم/القيعان، وعتبته تُضبط تلقائياً حسب تذبذب السهم (ATR).
2. لكل تسلسل نقاط، يُفحص ضد **قوانين النمط الإلزامية** (مثل: W4 لا تتداخل مع W1).
3. تُحسب **درجة الثقة** = 60% قوانين + 40% قرب نسب فيبوناتشي من أهدافها.
4. الأنماط المحققة لكل القوانين تُرتّب حسب الدرجة، مع **أهداف ووقف خسارة**.

## ضبط الدقة (مهم)
- `--pct 0.05` : عتبة ZigZag يدوية. أكبر = موجات أكبر/أقل عدداً (للأسهم المتقلبة).
- `--atr-mult 2.0` : أو اترك العتبة تلقائية لكن غيّر المضاعف.
- `--min-score 70` : للفرص عالية الثقة فقط.

## تطويرات الدقة (مطبّقة ✅)
كل نمط يحصل على **درجة بنية** (قوانين+فيبو) و**درجة نهائية** بعد 5 تأكيدات:

**للأنماط الحافزة (Impulse/Diagonal):**
1. **حجم W3 > W1** (ص21)
2. **ماكد W3 الأقوى** منذ بداية الموجة (ص18)
3. **دايفرجنس الماكد** بين W3 و W5 لإنهاء الموجة (ص19)
4. **كسر W3 للقناة** الموجية (ص20)
5. **توافق الفريم الأكبر (MTF)** — اتجاه ~100 يوم (ص66)

**للأنماط التصحيحية (Zigzag/Flat/Triangle):**
- التصحيح عكس الاتجاه الأكبر (انضمام صحي للترند)
- حجم A > C في الزجزاج (ص39)
- قرب السعر من متوسط 200

> الدرجة النهائية = البنية × (0.70 + 0.30 × نسبة التأكيدات).

### Backtest — قياس الجدوى تاريخياً
```bash
python backtest.py ./data --rr 2 --max-hold 40 --min-struct 55
python backtest.py ./data --min-confirm 0.6      # صفقات مؤكَّدة فقط
```
يعطي **نسبة الفوز** و**العائد بوحدة R** لكل نوع نمط. (النتائج التجريبية:
Flat/Zigzag/Impulse رابحة، Triangle خاسر — مطابق لنصيحة ص67).

### رسم الموجات
```bash
python plot_waves.py ./data --ticker AAPL
python plot_waves.py ./data --ticker LRCX --pattern Impulse --out lrcx.png
```
يرسم الشارت مع تأشير 0-1-2-3-4-5 (أو A-B-C / A-E) وخطوط الأهداف.

## ملاحظة على طبيعة إليوت
النظرية **كسرية (fractal)** — نفس النقاط قد تُفسَّر كنمط داخل نمط (مثلاً ساق
داخل الدافعة قد تظهر كـ Zigzag مستقل). لذلك يُستحسن دائماً مقارنة **الدرجة
الموجية** على أكثر من فريم زمني قبل القرار، تماماً كما في "اسأل نفسك" (ص66).
